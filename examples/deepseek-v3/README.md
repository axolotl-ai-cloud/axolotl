# DeepSeek-V3 Mini: Larger MoE Examples

This folder contains example configs for training larger randomly-initialized DeepSeek‑V3‑style MoE models generated by the helper script.

Models are created locally with random init using:

```
python scripts/make_deepseek_v3_mini.py --params 3b --out ./models/deepseek-v3-mini-3b
python scripts/make_deepseek_v3_mini.py --params 7b --out ./models/deepseek-v3-mini-7b
```

Then train with the corresponding example YAML:

```
accelerate launch -m axolotl.cli.train examples/deepseek-v3/deepseek-v3-3b-moe.yml
accelerate launch -m axolotl.cli.train examples/deepseek-v3/deepseek-v3-7b-moe.yml
```

Notes
- moe_kernels: true enables the CG‑GEMM MoE kernels; on small batches they may be neutral or slower. Larger per‑expert M (bigger micro_batch_size and/or sequence_len) shows clearer gains.
- You can reduce padding overhead for small batches by setting `AXOLOTL_MOE_GROUP_SIZE_M=64`.
- For deeper profiling, set `AXOLOTL_LOG_LEVEL=DEBUG` and optionally `AXOLOTL_MOE_VERBOSE=1`.
- Tokenizer: Examples reference `tokenizer_config: axolotl-ai-co/DeepSeek-V3-1B`. Ensure network access for the first run, or copy that tokenizer into your local model directory.
